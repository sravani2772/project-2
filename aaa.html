AAA testing with Python

WHAT IS A AAA TESTING?

Arrange/Act/Assert (AAA) is a pattern for arranging and formatting code in Unit Test methods.

It is a best practice to author your tests in more natural and convenient way. The idea is to develop a unit test by following these 3 simple steps:

Arrange – setup the testing objects and prepare the prerequisites for your test.

Act – perform the actual work of the test.

Assert – verify the result.


BENEFITS OF USING ARRANGE ACT ASSERT

1. Distinguishes what's being tested from the setup and verification procedures.

2. Clarifies and concentrates attention on a set of test steps that has proven to be successful in the past and are generally required.

3. There were assertions jumbled in with the "Act" code.

4. Methods of testing that attempt to test too many things at once.



Each step in the pattern is a function whose return value is the input of the following step. For example,

assert(act(arrange()));

Each step is represented by a function that is called from left to right. The first command is to arrange. The arrange function returns a value, which is then provided to the act function, which returns a value, which is then sent to assert. Assert returns void, a bool, or any value you, the author, deem useful for the test.

ARRANGE

A unit test application's initial step is to create a test case. We'll set up the test here, or in other words, we'll conduct the essential setup. For example, in order to run the test, we must first construct an object of the desired class, then build dummy objects and other variable initialization, and so on.

Example: fruit = ['apples','bananas','Calabash','Dragonfruit']



ACT

The middle step of a unit step application is this. We will run the test in this stage. To put it another way, we'll do the real unit testing and get the results from the test application. Basically, we'll use the object we built in the previous step to call the targeted function in this step.

Example: result = fruit.reverse()



ASSERT

A unit test application's final step is to run it. We'll compare the returned result to the expected outcomes in this phase.

Example: assert result is None

assert fruit == ['apples','bananas','Calabash','Dragonfruit']



OUTPUT:

def test_reverse():

"""

list.reverse inverts the order of items in a list, in place

"""

fruit = ['apples','bananas','Calabash','Dragonfruit']

result = fruit.reverse()

assert result is None

assert fruit == ['apples','bananas','Calabash','Dragonfruit']
